# AIDE API Documentation

## Overview

The AIDE service is a supporting service in the Codai ecosystem, providing essential functionality for the platform.

## Base URL

- **Development**: `http://localhost:4001`
- **Production**: `https://AIDE.codai.dev`

## Authentication

All API endpoints require authentication via JWT tokens or API keys.

### Headers

```
Authorization: Bearer <jwt_token>
Content-Type: application/json
X-API-Key: <api_key>
```

### GET /metrics

Returns Prometheus metrics for monitoring.

## API Endpoints

### Core Endpoints

#### GET /api/v1/AIDE

Get AIDE data.

**Parameters:**

- `limit` (optional): Number of items to return (default: 10)
- `offset` (optional): Number of items to skip (default: 0)

**Response:**

```json
{
  "data": [],
  "pagination": {
    "limit": 10,
    "offset": 0,
    "total": 100
  }
}
```

#### POST /api/v1/AIDE

Create new AIDE record.

**Request Body:**

```json
{
  "name": "string",
  "description": "string",
  "metadata": {}
}
```

**Response:**

```json
{
  "id": "uuid",
  "name": "string",
  "description": "string",
  "metadata": {},
  "created_at": "2024-01-01T00:00:00.000Z",
  "updated_at": "2024-01-01T00:00:00.000Z"
}
```

#### GET /api/v1/AIDE/:id

Get specific AIDE record by ID.

**Response:**

```json
{
  "id": "uuid",
  "name": "string",
  "description": "string",
  "metadata": {},
  "created_at": "2024-01-01T00:00:00.000Z",
  "updated_at": "2024-01-01T00:00:00.000Z"
}
```

#### PUT /api/v1/AIDE/:id

Update specific AIDE record.

#### DELETE /api/v1/AIDE/:id

Delete specific AIDE record.

## Error Responses

All endpoints return standardized error responses:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human readable error message",
    "details": {},
    "timestamp": "2024-01-01T00:00:00.000Z",
    "request_id": "uuid"
  }
}
```

### Error Codes

- `400` - Bad Request
- `401` - Unauthorized
- `403` - Forbidden
- `404` - Not Found
- `409` - Conflict
- `429` - Rate Limited
- `500` - Internal Server Error
- `503` - Service Unavailable

## Rate Limiting

- 100 requests per minute per API key
- 1000 requests per hour per API key
- Burst capacity: 200 requests

## SDK Examples

### JavaScript/TypeScript

```typescript
import { CodaiClient } from '@codai/sdk';

const client = new CodaiClient({
  baseURL: 'https://AIDE.codai.dev',
  apiKey: 'your-api-key',
});

// Get AIDE data
const data = await client.AIDE.list({ limit: 20 });

// Create new record
const newRecord = await client.AIDE.create({
  name: 'Example',
  description: 'Example description',
});
```

### Python

```python
from codai_sdk import CodaiClient

client = CodaiClient(
    base_url='https://AIDE.codai.dev',
    api_key='your-api-key'
)

# Get AIDE data
data = client.AIDE.list(limit=20)

# Create new record
new_record = client.AIDE.create({
    'name': 'Example',
    'description': 'Example description'
})
```

## Monitoring

### Metrics

The service exposes the following metrics:

- `http_requests_total` - Total HTTP requests
- `http_request_duration_seconds` - Request duration histogram
- `AIDE_operations_total` - Service-specific operations
- `AIDE_errors_total` - Service-specific errors

### Logs

```json
{
  "timestamp": "2024-01-01T00:00:00.000Z",
  "level": "info",
  "service": "AIDE",
  "message": "Request processed",
  "request_id": "uuid",

  "duration": 123
}
```

### Tracing

Distributed tracing is available via Jaeger with the following tags:
